# Output Patterns

## New Draft

Use:

```text
## 完成稿
[Japanese ES]

## 構成
- 冒頭: ...
- 根拠: ...
- 貢献: ...

## さらに強くするために
- ...
```

## Revision

Use:

```text
## 修正版
[Japanese ES]

## 主な修正点
- ...

## 原文で弱かった点
- ...
```

## Chinese Notes For The User

When the user is Chinese-speaking, add short Chinese notes if helpful:

```text
中文说明：
这版把重点从“我很感兴趣”改成了“我有相关经历，所以能在这个岗位贡献”。日本 ES 更重视具体行动和入社后的贡献。
```

## Multiple Versions

When the user asks for variants, produce distinct versions:

- Standard version: balanced and safe.
- Stronger version: sharper contribution and company fit.
- Short version: compressed for strict limits.
- Natural version: smoother Japanese, less formal.

## Character Limit

If the user gives a limit:

- Mention approximate character count.
- Prefer preserving concrete action and company fit over decorative wording.
- If the request requires exact counting, use a local script or command to count Japanese characters.
